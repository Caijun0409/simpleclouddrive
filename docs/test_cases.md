# 最小功能验证用例集

## TC-001 首页展示

前置条件：

- App 已安装并可启动。
- 首次启动或已完成 mock 数据初始化。

操作步骤：

1. 启动 App。
2. 停留在底部导航“网盘”页。
3. 查看用户信息卡片、最近转存、最近浏览区域。

预期结果：

- 显示昵称“训练营用户”。
- 显示文件总数。
- 显示已使用容量，单位为 B、KB、MB 或 GB。
- 最近转存为空时显示“暂无转存记录”。
- 最近浏览为空时显示“暂无浏览记录”。

## TC-002 文件列表展示

前置条件：

- App 已启动。
- mock 数据已初始化。

操作步骤：

1. 点击底部“文件”tab。
2. 查看根目录文件列表。

预期结果：

- 能看到 `Documents`、`readme.txt`、`training.mp4` 等 mock 文件。
- 每个文件显示图标、名称、大小或文件夹标识、修改时间。
- 每个文件右侧有更多菜单。

## TC-003 文件夹进入

前置条件：

- 位于文件列表根目录。
- 存在 `Documents` 文件夹。

操作步骤：

1. 点击 `Documents`。
2. 查看当前页面标题和面包屑。
3. 点击系统返回或顶部返回按钮。

预期结果：

- 进入 `Documents` 文件夹。
- 列表显示该文件夹下文件，例如 `notes.txt`。
- 顶部可返回上一级。
- 返回后回到根目录。

## TC-004 上传

前置条件：

- 设备 Downloads 中有 `test.txt` 或其他普通文件。
- App 有可用文件选择器。

操作步骤：

1. 进入“文件”tab。
2. 点击顶部“上传”。
3. 在系统文件选择器中选择 `test.txt`。
4. 返回 App 后查看当前目录。
5. 回到首页查看最近转存。

预期结果：

- 可以从 Downloads 中选择 `test.txt`。
- 上传成功后显示 Snackbar“上传成功”。
- 文件出现在当前目录。
- 首页“最近转存”显示该文件。
- 数据库中的 path 保存 App 私有目录下的真实绝对路径。

## TC-005 TXT 阅读器

前置条件：

- 文件列表中存在 `readme.txt` 或已上传 TXT 文件。

操作步骤：

1. 点击 `readme.txt`。
2. 查看阅读器标题、正文和页码。
3. 左滑。
4. 右滑。

预期结果：

- 进入 TXT 阅读器。
- 能看到真实中文文本内容。
- 页码不是 `0 / 0`。
- 左滑进入下一页，右滑返回上一页。
- 第一页和最后一页不会越界。

## TC-006 最近浏览

前置条件：

- 存在可打开的 TXT 文件。

操作步骤：

1. 在文件列表打开一个 TXT 文件。
2. 返回首页。
3. 查看“最近浏览”区域。

预期结果：

- 最近浏览中显示刚打开的 TXT 文件。
- 多次打开同一个文件时，该文件仍只出现一条，时间被更新到最新。

## TC-007 最近转存

前置条件：

- 设备中存在可上传文件。

操作步骤：

1. 上传一个文件。
2. 返回首页。
3. 查看“最近转存”区域。

预期结果：

- 最近转存显示上传文件。
- 列表按最近上传时间倒序。
- 最多展示 10 条。

## TC-008 重命名

前置条件：

- 文件列表中存在任意文件。

操作步骤：

1. 点击文件右侧更多菜单。
2. 选择“重命名”。
3. 输入新名称并确认。

预期结果：

- 文件名更新。
- 修改时间更新。
- 真实物理文件路径不改变。
- 输入空名称时显示失败提示，不应崩溃。

## TC-009 移动

前置条件：

- 根目录存在至少一个文件和 `Documents` 文件夹。

操作步骤：

1. 点击某个文件右侧更多菜单。
2. 选择“移动”。
3. 在目标文件夹 Dialog 中选择 `Documents`。
4. 进入 `Documents` 查看。
5. 再将文件移动回根目录。

预期结果：

- 文件从原目录消失。
- 文件出现在目标目录。
- 移动后列表自动刷新。
- 文件夹不能移动到自己或自己的子孙目录。

## TC-010 删除

前置条件：

- 文件列表中存在可删除文件。
- 若验证递归删除，需要准备一个包含子文件的文件夹。

操作步骤：

1. 点击文件右侧更多菜单。
2. 选择“删除”。
3. 在确认框中确认删除。
4. 若删除文件夹，进入原位置检查子文件是否还存在。

预期结果：

- 删除成功后文件从列表消失。
- App 私有目录文件被删除。
- recent_browse 和 recent_transfer 中关联记录被清理。
- 删除文件夹时，子文件和子文件夹一起删除。

## TC-011 分享 DeepLink

前置条件：

- 文件列表中存在任意文件。
- 设备连接 adb。

操作步骤：

1. 点击文件右侧更多菜单。
2. 选择“分享”。
3. 复制 Dialog 中的链接。
4. 确认链接格式为 `simplecloud://share?shareId=xxx`。
5. 执行 adb 命令：

```bash
adb shell am start -W -a android.intent.action.VIEW -d "simplecloud://share?shareId=xxx" com.example.simpleclouddrive
```

6. 使用一个不存在的 shareId 再执行一次。

预期结果：

- 分享链接只包含 shareId，不包含文件名、路径、大小。
- 有效 shareId 能拉起 App 并进入分享文件列表页。
- 分享文件列表展示被分享文件。
- 无效 shareId 显示“分享链接无效”。

## TC-012 视频系统播放器

前置条件：

- 上传一个真实 mp4 文件。
- 设备中存在可打开视频的系统播放器。

操作步骤：

1. 在文件列表点击上传后的 mp4。
2. 等待系统播放器拉起。
3. 返回 App。
4. 进入首页查看最近浏览。

预期结果：

- 系统播放器被拉起。
- App 没有暴露 `file://` Uri。
- 返回 App 后最近浏览显示该视频。
- 如果设备没有播放器，显示“未找到可打开该视频的应用”。
